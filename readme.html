User Scripts


Show the live status of linked OPS and Jira requests directly in the FortiCare ticket lists, as colour-coded badges with a time-since-last-activity chip.

Before you start


These two scripts are standalone — they do not require the old FortiCare Ticket List script. They read status using your existing browser session, so:

  • Be signed in to ops.fortisase.com for the OPS script.
  • Be signed in to the Jira Service Management portal for the Jira script.
  • On first run, when Tampermonkey asks to access a cross-origin domain, choose “Always allow domain”.

A badge looks like this — refresh control, status, then time since last activity:  ↻OP510288 · OPEN20m  ↻EC7928 · IN PROGRESS3h

Installation Tutorial

These user scripts are compatible with the Tampermonkey extension. Tampermonkey is the most popular userscript manager, with over 10 million users.

If you still use the Greasemonkey extension with Firefox, please uninstall it (it is no longer supported since Firefox 57) and install Tampermonkey instead.

The Tampermonkey extension and these userscripts have been tested with:

  • Chrome / Brave browsers
  • Firefox

Tampermonkey makes it easy to manage your userscripts and provides a clear overview of running scripts, a built-in editor, import/export and automatic update checks.

Instructions for Chrome

  • Open Chrome and go to the Tampermonkey web store page.

  • Click on “Add to Chrome”


  • Click on “Add extension”


  • Tampermonkey is now installed on your Chrome browser


  • In the Scripts table above, click the script’s Install link (forticare_ops.user.js or FortiCare_Jira.user.js), then click “Install” in Tampermonkey.


  • On first run, when the script requests access to a cross-origin resource (ops.fortisase.com or the Jira portal), click “Always allow domain”.

Instructions for Firefox

  • Open Firefox and go to the Tampermonkey add-on page.

  • Click on “Add to Firefox”


  • Click on “Add”


  • Click on “OK”


  • Tampermonkey is now installed on your Firefox browser


  • In the Scripts table above, click the script’s Install link, then click “Install” in Tampermonkey.


  • On first run, when the script requests access to a cross-origin resource, click “Always allow domain”.
